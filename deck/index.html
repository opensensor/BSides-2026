What is firmware? What is a SoC?

// the terrain we're about to map
Firmware

Persistent software that runs the device itself

  • Lives on flash chips — SPI NOR, SPI NAND, eMMC
  • Layered System: bootloader → kernel → rootfs
  • Readable, modifiable, reflashable — usually
  • Its what actually runs when you plug the thing in
System on Chip

CPU + RAM + peripherals on one die

⤢ click to expand
  • ISP, NPU, crypto engines, GPIO, video encoders
  • Contains a boot ROM — immutable mask ROM
  • OTP fuses carry per-chip secrets & config
  • What runs first is fixed in silicon at fab time — not by you.

$ man flash --types § 02 · flash storage
Chapter 02 · Know your flash

NOR vs NAND — not the same chip

// the chip you clip onto changes everything downstream
SPI NOR
byte-addressable
⤢ click to expand
  • Small — 8, 16, or 32 MB typical
  • Execute-in-place (XIP) capable
  • No bad blocks — what you read is what's there
  • Filesystem: SquashFS or raw blobs
SPI NAND
page / block-addressable
⤢ click to expand
  • Big — 128 MB to multi-GB
  • Read in pages, erase in blocks
  • Born with bad blocks — and grows more over time
  • Filesystem: UBI + UBIFS over MTD
More space — more abstraction
NAND density is paid for in software: the kernel runs UBI — wear leveling, bad-block tracking, ECC — underneath UBIFS. Your filesystem is layers above raw silicon.
Why it matters to us
Dumping NAND is not dumping a filesystem — you get raw pages with OOB ECC bytes. Reconstructing the rootfs means decoding UBI volumes before you ever see /etc.

Anatomy

ioctl, deconstructed

// _IOR / _IOW + struct + dispatch in fops->unlocked_ioctl
drivers/tx-isp/tx-isp.c
/* the contract — what userspace can ask for */ #define TX_ISP_MAGIC 'I' #define TX_ISP_GET_FRAME _IOR (TX_ISP_MAGIC, 0x01, struct isp_frame) #define TX_ISP_SET_TUNING _IOW (TX_ISP_MAGIC, 0x02, struct isp_tune) #define TX_ISP_QUERY_STATS _IOWR(TX_ISP_MAGIC, 0x03, struct isp_stats) /* the dispatch — kernel side */ static long tx_isp_ioctl(struct file *f, unsigned int cmd, unsigned long arg) { switch (cmd) { case TX_ISP_GET_FRAME: return isp_get_frame(f, arg); case TX_ISP_SET_TUNING: return isp_set_tuning(f, arg); case TX_ISP_QUERY_STATS: return isp_query_stats(f, arg); } return -ENOTTY; } static const struct file_operations tx_isp_fops = { .owner = THIS_MODULE, .unlocked_ioctl = tx_isp_ioctl, // THE entry point .open = tx_isp_open, };
01 · Macro

_IOR / _IOW / _IOWR packs direction, magic, nr, and size into a single 32-bit cmd.

02 · Contract

The struct is the wire format. Kernel and user must agree byte-for-byte.

03 · Dispatch

A switch in unlocked_ioctl. Each case is one endpoint — find this and you've found the whole API.

Tools of the trade

Reverse engineering kernel modules

// MIPS · stripped symbols · undocumented register banks
Binary Ninja

Strong MIPS HLIL, scriptable, fast

  • HLIL reads close to C — great for ioctl dispatch tables
  • Python plugin API for batch annotation
  • Type system that survives across binaries
  • Commercial — license cost is the friction
Ghidra

Free, NSA-blessed, decompiles MIPS

  • P-code IR — cross-arch queries
  • Strong batch / headless mode
  • Decent kernel-module loader after some setup
  • Slower UX, but unbeatable price
What helps most
Loading the matching vmlinux alongside the .ko so kernel symbol references resolve. Without it, every call into the kernel is a black hole.
What still hurts
Indirect calls through function pointers in file_operations tables. Static analysis loses the trail; runtime tracing fills the gap.

Chapter 05 · What we found

Two vulnerability classes

Bug class 01 · T31

32-bit verification

  • Verifier checks one 32-bit word of the RSA output
  • RSA-2048 / SHA-256 reduced to a 32-bit brute force
  • ~217 seconds to forge an image
  • The crypto is there — just barely enforced.
Bug class 02 · T32 / T40 / T41 / A1

Init-table bypass

  • ROM parses an init table before verifying the signature
  • Init table supports full 32-bit write addresses
  • Write to SRAM → corrupt secure-boot state → skip verify
  • The verification is sound. It just never runs.
→ Different chips. Different flaws. Same outcome: unsigned code runs. coordinated disclosure in progress
